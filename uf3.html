1) Tipos de diseño web (examen)

Fijo Elástico Fluido Responsivo
Regla de oro: memoriza la unidad típica:
  • Fijo → px (no se adapta)
  • Elástico → em (depende del texto)
  • Fluido → % (depende del ancho)
  • Responsivo → @media (cambia según pantalla)

Fijo (estático)

  • Medidas fijas en px. En móvil puede aparecer scroll horizontal.
  • Ventaja: simple. Desventaja: mala UX en pantallas pequeñas.
.contenedor { width: 1024px; margin: 0 auto; }

Elástico

  • Usa em: escala con el tamaño del texto.
  • No es responsive por pantalla: es “responsive al texto”.
h1 { font-size: 2em; width: 12em; }

Fluido (líquido)

  • Usa %: se adapta al ancho del viewport.
  • Problema típico: en pantallas muy grandes el texto puede quedar demasiado ancho.
.col { width: 50%; }

Responsivo (adaptable)

  • Usa media queries para aplicar CSS distinto.
  • Concepto clave: breakpoints.
  • Enfoque muy típico: mobile-first.
@media screen and (max-width: 768px) { .menu { display:none; } }

Lo que suelen preguntar

  • Reconocer diseño por unidades (px / em / %).
  • Ventaja/desventaja de fijo vs fluido.
  • Qué significa mobile-first.
  • Qué técnica hace responsive (media queries).
Trampa típica: “Elástico” no depende de la pantalla; depende del tamaño del texto.

2) Media Queries (examen)

@media screen/print/all min/max-width orientation resolution
Definición: reglas CSS condicionales. Si se cumple la condición, se aplica el bloque.

Meta viewport (móvil)

<meta name="viewport" content="initial-scale=1, width=device-width">

Sintaxis base

@media screen and (max-width: 768px) {
  /* CSS que solo se aplica si se cumple la condición */
}

Medios

  • screen (pantalla), print (impresión), speech, all (por defecto).
  • Trampa: si no indicas medio, es all.

Operadores

  • and → todas las condiciones
  • , (coma) → OR
  • not → niega
  • only → compatibilidad con navegadores antiguos

Propiedades típicas

  • min-width, max-width (las más importantes)
  • orientation: portrait | landscape
  • resolution (dpi, dppx, dpcm; más usado en print)

Breakpoints de ejemplo

/* móvil */
@media screen and (max-width: 767px) { }

/* tablet */
@media screen and (min-width: 768px) and (max-width: 1279px) { }

/* escritorio */
@media screen and (min-width: 1280px) { }

Trampas de test

  • Condición SIEMPRE entre paréntesis: (max-width: 768px).
  • No existe (width < 768px) en CSS.
  • La coma es OR: @media screen, print.
  • Sin medio → all (no “solo screen”).
Ejemplo de OR con coma 
@media screen and (max-width: 600px), print {
  body { font-size: 14px; }
}
Ejemplo de orientación 
@media (orientation: landscape) {
  body { background: #222; }
}

3) Efectos avanzados (examen)

3.1 Sombras

Texto → text-shadow · Caja → box-shadow 
/* text-shadow: x y blur color */
h1 { text-shadow: 2px 2px 4px #000; }

/* box-shadow: [inset] x y blur spread color */
.card { box-shadow: 8px 8px 10px 0 rgba(0,0,0,.4); }
.card2 { box-shadow: inset 4px 4px 6px 0 rgba(0,0,0,.4); }
  • inset → sombra interior.
  • spread agranda/reduce la sombra (no confundir con blur).
  • Múltiples sombras separadas por comas.

3.2 Gradientes

Regla: un gradiente necesita mínimo 2 colores. 
/* lineal (por defecto: to bottom) */
.bg1 { background: linear-gradient(red, blue); }

/* radial (tamaño por defecto: farthest-corner) */
.bg2 { background: radial-gradient(circle, red, blue); }

/* cónico */
.bg3 { background: conic-gradient(from 30deg, blue, beige, pink, green); }

/* varios gradientes con comas (se superponen) */
.bg4 { background:
  radial-gradient(rgba(255,255,255,0), rgba(255,255,255,.6)),
  linear-gradient(45deg, blue, red);
}

3.3 Transiciones

Clave: necesita 2 estados (p.ej. normal y :hover). La duración es obligatoria. 
.btn {
  background: #1e90ff;
  transition: background-color 1.5s ease, transform .5s linear;
}
.btn:hover {
  background: beige;
  transform: scale(1.1);
}

3.4 Transformaciones

Trampa: transform NO cambia el flujo del documento; solo la apariencia. 
.caja { transform: scale(1.2) translate(20px) rotate(10deg); }
/* El orden cambia el resultado */
  • scale() tamaño, translate() mueve, rotate() gira, skew() inclina.
  • Se combinan y el orden importa.

Checklist rápido (3.x)

  • text-shadow orden: x, y, blur, color.
  • box-shadow incluye opcional inset y spread.
  • linear-gradient por defecto: to bottom.
  • radial-gradient tamaño por defecto: farthest-corner.
  • transition se declara en el estado base (no en :hover).
  • transform: el orden importa.

4) Preprocesadores (SASS/SCSS/LESS) (examen)

Idea clave: el navegador solo entiende CSS.
SASS/SCSS/LESS deben compilarse a CSS.

4.1 Procesadores CSS

  • Preprocesador: genera CSS desde un metalenguaje.
  • Postprocesador: optimiza el CSS ya generado (p.ej. PostCSS).
  • Lo habitual: se complementan.
SCSS/LESS → (compilar) → CSS → (PostCSS) → CSS optimizado

4.2 Tipos

  • SASS (sintaxis .sass): sin llaves ni ;, sangrado estricto.
  • SCSS (sintaxis .scss): parecida a CSS; un CSS válido es SCSS válido.
  • LESS: variables con @; compilación en cliente o servidor.
/* SCSS */
$color: red; h1 { color: $color; }

/* LESS */
@color: red; h1 { color: @color; }

/* SASS */
$color: red
h1
  color: $color

4.3 Elementos

  • Variables (valores)
  • Operadores (+ - * / %)
  • Anidación y & (selector padre)
  • Mixins (reutilizar código)
  • Funciones (devuelven valores)
  • Directivas: @import, @extend, @if, @for...
  • @for ... through incluye el final; to no.
Ejemplo corto integrando todo (SCSS) 
$color: #1e90ff;

@mixin sombra($b: 10px) {
  box-shadow: 0 0 $b rgba(0,0,0,.35);
}

@function doble($n) { @return $n * 2; }

.boton {
  color: $color;
  padding: doble(6px);
  @include sombra(12px);

  &:hover { background: $color; color: white; }
}

Trampas de test (4.x)

  • SCSS es sintaxis de SASS (no herramienta distinta).
  • LESS usa @ para variables (ojo con confundir con @media).
  • Mixin ≠ función: mixin inserta código; función devuelve valor.
  • through incluye final; to lo excluye.
Trampa típica: “El navegador interpreta SCSS/LESS”. → FALSO.
